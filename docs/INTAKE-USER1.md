# Intake — User #1 (Daimon / StackPenni)

*Repo location: `docs/INTAKE-USER1.md`. Everything the human provides, gathered in ONE session (~60–90 min, mostly gathering files and talking). Rule: after this session, the system never asks the person for materials again — only for reactions and gate decisions. This checklist doubles as the template for any future user's intake.*

## 1. Business profile (~10 min, spoken Q&A)
- [ ] What the business is, in your own words (talk, don't write)
- [ ] Brands: StackPenni + Island Futurist, Digital Sou-Sou, Caribbean Receipts — what each one is for
- [ ] Core subjects (the system turns these into the tag taxonomy)
- [ ] Platforms to publish on, and which matters most
- [ ] Goals for the next 6 months (what does winning look like?)
- [ ] Red-lines: topics, stances, or styles the brand never touches

## 2. Voice materials (target: 2,000+ of YOUR words, or 30+ min of speech)
Any combination — natural speech weighted highest:
- [ ] WhatsApp chat export(s) where you talk a lot (the system strips everyone else's messages)
- [ ] Emails you wrote (sent folder selects; forwarded/quoted text is stripped)
- [ ] Voice notes — existing ones, or record fresh ones in the session
- [ ] Your messages from AI chats (e.g., this conversation's your-side text), social posts, anything you authored
- [ ] If thin: the console runs the 10–12 question spoken interview to fill the gap

## 3. Seed sources (10–30 items)
- [ ] URLs / RSS feeds / newsletters you actually read and trust for this business
- [ ] YouTube channels + social accounts worth monitoring
- [ ] Documents & notes that represent "our kind of material" (OB1 exports welcome)
- [ ] 3–5 anti-examples: sources or pieces you consider junk for this business
*(The existing ~1,545-source bank gets re-scored automatically — no action needed.)*

## 4. Taste examples
- [ ] 5–10 links to content in the domain you admire ("wish we'd made this") — any platform
- [ ] 3–5 links to content you hate ("the slop we never make")
- [ ] 2–3 stories you tell often (record them as voice notes — these feed Story Frameworks AND the voice corpus)

## 5. Visual materials
- [ ] 20+ phone photos/clips from your world: streets, markets, receipts, hands, screens, sea — imperfect is the point
- [ ] Brand assets if any (logo, colors) — "none yet" is a valid answer
- [ ] 3–5 examples of visual styles you like (screenshots/links)

## 6. First seeds (fuel for the co-production sprint)
- [ ] 3–5 voice notes, ~30 seconds each: an opinion, a story, a take on something current in Caribbean money culture. Messy and spoken. These become pieces 1–5.

## 7. Operational (you do these directly — credentials NEVER go to the AI or into the repo)
- [ ] Connect social accounts to Postiz yourself, in Postiz's own interface
- [ ] Create the YouTube Data API key yourself; place it on the VPS as an environment variable (Hermes documents the exact steps; you paste the key only into the server env, nowhere else)
- [ ] Pick your weekly gate day + time (~20 min, e.g., Sunday morning)

## What happens after this session
The playbook chain runs on everything above (Business Profile → Voice → Sources → Viral Patterns → Audience → Story → Format → Visual). You'll be called back only for **calibration and gates**: pick the voice sample that sounds like you, confirm the source criteria, approve the pattern starters. Then the co-production sprint begins with your first seed.

## Format notes
- Voice notes: any common audio format; the console transcribes
- Chat exports: WhatsApp "export chat without media" .txt is fine
- Don't clean anything up. Raw is signal; polished is noise.
