# PROPOSED AMENDMENT — Visual Style Module — Fictional Recurring Persona

**Status:** PROPOSED — awaiting operator gate approval (per CORRECTION-episode-format-and-reference-assets-v1.0 §1.4)
**Date:** 2026-07-17
**Source:** CORRECTION-episode-format-and-reference-assets-v1.0 §1.4
**Applies to:** modules/stackpenni/visual-style.md → Blend rules section

---

## What this amendment adds

Add the following rule to the `## Blend rules` section of `modules/stackpenni/visual-style.md`, as a new `###` entry after the existing "Disclosure" subsection:

> **Fictional recurring persona (episode format):** The show's narrator character is an openly fictional, AI-generated persona — a storytelling device, not a depiction of any real person. Episodes using the persona follow platform AI-disclosure rules (fully generated visuals → platform AI label). The persona never makes first-person claims about real events, real results, or real partnerships; those remain real-footage anchors. "Never present a generated visual as a real person" stands unchanged.

## Why

This is the resolution of finding F-G from the correction: the rule protecting trust stays intact; the parable device is added beside it, gated. A recurring AI narrator character is a *fictional, disclosed persona* — a different thing from faking a real person — but introducing it is a module change and therefore must pass the module review gate with the operator's explicit approval.

## What does NOT change

- "Never present a generated visual as a real person" — stands unchanged
- Real anchors (real people, real locations, real products, real results) — still require real footage
- Platform AI-disclosure rules — still apply, and are strengthened for this format (fully generated visuals → platform AI label)

## Approval gate

This amendment must be approved through the module review gate. Do not apply it to `modules/stackpenni/visual-style.md` until the operator has explicitly approved it.
