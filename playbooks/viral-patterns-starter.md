# Playbook: Viral Patterns Starter

*Repo location: `playbooks/viral-patterns-starter.md` · Executed by the system's AI during onboarding, through the console. v1.0*

<!-- run_order: 4 -->
<!-- display_label: Viral Patterns -->

## Purpose

Seed the Viral Patterns Playbook module at onboarding so the outward loop starts from a v1, not from empty.

## Inputs

5–10 links the user admires in their domain ("I wish we'd made this") + 3–5 anti-examples ("this is the slop we never make") + top performers pulled from the monitored channels (Sources Engine output).

## Procedure

1. Pull/transcribe each example.
2. AI analyzes per item: hook type, structure, emotional beat, format, pacing, why it likely worked — **framed as hypothesis, never fact**.
3. Cluster into named patterns with examples.
4. Anti-examples become the "never" list.
5. Gate → module v1.

## Output

Viral Patterns Playbook v1 (patterns + evidence + hypothesis framing + never-list).

## Gate

User confirms patterns ring true for their taste. Updated forever after by the two learning loops.
